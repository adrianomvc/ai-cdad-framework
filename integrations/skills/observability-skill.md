# Observability Skill

## Purpose

Validate logs, metrics, traces, alerts and dashboards.

## When to Use

- Entry Lane: Operational or Engineering
- Risk Modes: HYBRID/SAFE
- Typical triggers: incident, production change, rollout, SLO/SLA concern

## Steps

1. Identify critical signals.
2. Check logs without copying sensitive data.
3. Check metrics.
4. Check traces.
5. Check alerts.
6. Define post-deploy monitoring.
7. Update observability-checklist.md.

## Stop Conditions

Stop if sensitive data appears in logs or alert changes require approval.
