# Data Quality Skill

## Purpose

Validate data quality rules, evidence and impact.

## When to Use

- Entry Lane: Product or Engineering
- Risk Modes: HYBRID/SAFE
- Typical triggers: data quality rule, dataset validation, pipeline quality issue

## Steps

1. Identify dataset/table/pipeline.
2. Identify quality dimension.
3. Identify rule definition.
4. Validate sample/evidence.
5. Assess downstream impact.
6. Define validation metrics.
7. Create validation-report.md.

## Related MCPs

- Data Catalog MCP
- Observability MCP
- Cloud MCP, if approved
