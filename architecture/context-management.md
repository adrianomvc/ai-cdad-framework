# Context Management

## Princípio

Guardar bastante contexto nos repositórios, mas carregar pouco contexto na sessão do Devin.

## Camadas

- L0 Framework Context
- L1 Domain Context
- L2 Product Context
- L3 Initiative Context
- L4 Repo Execution Context

## Hot Context

Iniciativa atual, repo atual, artefatos da fase atual e runtime-context.md.

## Cold Context

Histórico arquivado, postmortems, SDDs antigos e decisões antigas. Não entra por padrão.

## Runtime

O arquivo principal da sessão Devin é `runtime-context.md`.
