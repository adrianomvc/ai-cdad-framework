# AI-CDAD Adoption Guide

## Recommended Path

1. Create personal GitHub repository.
2. Upload AI-CDAD framework package.
3. Run Devin prompt `00-plan-before-files.md`.
4. Review Devin's plan before creating or changing files.
5. Run scaffold.
6. Run architecture review.
7. Apply corrections.
8. Generate examples.
9. Run final audit.
10. Perform tabletop tests.

## Before moving to work environment

Validate:

- local bugfix flow
- operational incident flow
- HYBRID integration flow
- SAFE brownfield flow
- token minimization
- hard stops
- human approvals
- repository model

## Work Environment Rollout

1. Create central community repository.
2. Add AI-CDAD references.
3. Create product/domain context.
4. Add indexes.
5. Choose one pilot application repo.
6. Add `.cdad/`.
7. Run a local bugfix tabletop.
8. Run one real low-risk FAST scenario.
9. Iterate.
