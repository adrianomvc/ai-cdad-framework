# CDAD Decisions Log

## Core decisions

- PM Agent is external and runs outside CDAD.
- CDAD supports Product, Operational, Engineering and Compliance entry lanes.
- CDAD supports FAST, HYBRID and SAFE risk modes.
- CDAD governs; AWS AI-DLC executes.
- Devin is a guided orchestrator.
- PM defines WHAT / WHY / VALUE / PRIORITY.
- Tech Lead owns architecture, risk, contracts and rollout strategy.
- Developer owns execution and acts as AI-Augmented Engineer.
- AI drafts artifacts; humans validate or approve when required.
- Context must be minimized in runtime sessions.
- runtime-context.md is the primary runtime context.
- Full artifacts live in repositories; Devin sessions load only what is needed.
- Parallel execution is allowed only when safe, independent and validatable.
