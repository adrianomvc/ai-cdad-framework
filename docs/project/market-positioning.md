# Market Positioning

## Category

AI-CDAD fits into the emerging category of:

```text
AI-Augmented Delivery Governance
```

It combines aspects of:

- software delivery lifecycle governance
- AI-assisted engineering
- human-in-the-loop workflows
- context engineering
- agentic delivery orchestration
- platform engineering enablement
- enterprise architecture governance

## Why This Matters

The market is rapidly adopting AI coding agents and AI software engineers.

However, most teams still lack a safe operating model for:

- brownfield systems
- regulated environments
- data-sensitive applications
- multi-repo initiatives
- production-critical changes
- governance-heavy organizations

AI-CDAD addresses this gap.

## Differentiation

AI-CDAD differs from generic AI coding workflows because it is:

- risk-adaptive
- context-driven
- human-owned
- repository-based
- tool-agnostic
- governance-aware
- brownfield-safe
- evidence-oriented

## Comparable Concepts

AI-CDAD complements, but does not replace:

- SDLC
- AI-DLC
- SDD
- Agile delivery
- DevOps
- Platform Engineering
- Architecture Review Boards
- InnerSource governance
- Data Mesh governance

## Positioning Statement

For engineering organizations adopting AI agents, AI-CDAD is a delivery governance framework that enables faster execution while preserving human accountability, context control and production safety.
