# Adoption Model

## Phase 1 — Personal / Lab

Goal:

Validate the framework without exposing corporate information.

Actions:

- upload to personal GitHub
- run Devin planning prompt
- execute tabletop tests
- refine prompts and docs
- avoid any confidential data

## Phase 2 — Internal Pilot

Goal:

Adapt to a corporate environment.

Actions:

- create internal framework repo
- review with security/architecture
- create central community hub
- select one application repo
- install `.cdad/`
- run a FAST scenario

## Phase 3 — Squad Adoption

Goal:

Use AI-CDAD in daily work.

Actions:

- train Tech Leads and Developers
- define central hub owner
- maintain initiative registry
- use runtime-context and execution briefs
- review examples and playbooks

## Phase 4 — Governance Integration

Goal:

Integrate with enterprise systems.

Possible integrations:

- Jira
- GitHub / GitLab
- Confluence
- ServiceNow
- DataDog
- AWS
- internal data catalog
- MCP servers

## Phase 5 — Automation

Goal:

Move from documentation-first to workflow-assisted execution.

Possible automations:

- initiative scaffolding
- context retrieval
- policy checks
- hard stop detection
- artifact generation
- validation report generation
