# Use Cases

## 1. Local Bugfix

A developer finds a bug in an application repository.

AI-CDAD provides:

- Operational Lane
- triage
- FAST/HYBRID/SAFE classification
- runtime-context.md
- ai-execution-brief.md
- validation evidence

## 2. Production Incident

A production issue requires urgent action.

AI-CDAD provides:

- Operational Lane
- SAFE default when impact is critical
- hard stops
- rollback plan
- observability checklist
- approval record

## 3. Brownfield Feature

A feature changes existing production behavior.

AI-CDAD provides:

- impact analysis
- Tech Lead Design
- Lean SDD
- rollout and rollback
- contract validation
- regression validation

## 4. Greenfield Capability

A new isolated capability is created.

AI-CDAD provides:

- FAST mode
- lightweight governance
- focused execution brief
- developer review

## 5. Technical Debt / Refactor

A team improves internal structure without changing behavior.

AI-CDAD provides:

- Engineering Lane
- risk classification
- relevant-files-list.md
- validation plan

## 6. Compliance / Security Finding

A vulnerability or audit finding must be remediated.

AI-CDAD provides:

- Compliance Lane
- SAFE/HYBRID routing
- security approval
- evidence capture
- audit-friendly artifacts

## 7. Multi-Agent Tooling

A team uses Devin for autonomous work and Claude Code/Codex/Cursor for local work.

AI-CDAD provides:

- common runtime contract
- adapters
- generic execution brief
- consistent human gates
